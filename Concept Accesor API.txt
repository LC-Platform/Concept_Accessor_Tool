PDF Processing
POST /read-pdf/
Purpose: Upload a PDF and extract sections.
Request:
{
  "pdf": "multipart/form-data (UploadFile)"
}


Response:
{
  "chapter_id": "string",
  "section_ids": ["array of section IDs"],
  "message": "string"
}


Frontend Notes:
Send file as FormData
Store chapter_id for subsequent calls

Summary Generation
POST /full-summary/
Purpose: Generate AI-powered full chapter summary.
Request:
{
  "chapter_id": "string"
}


Response:
{
  "chapter_id": "string",
  "full_summary": "string OR array of strings"
}


Frontend Notes:
Use chapter_id from /read-pdf/
Display full_summary as text or list
POST /section-summary/
Purpose: Generate summary for a specific section.
Request:
{

  "section_id": "string"
}


Response:
{
  "chapter_id": "string",
  "section_id": "string",
  "section_summary": "string"
}


Frontend Notes:
Fetch summaries per section_id (e.g., "3.1")

Media Management
POST /generate-taxonomy-images/
Purpose: Generate taxonomy SVG diagrams for biology terms.
Request:
{
  "chapter_id": "string"
}


Response:
{
  "chapter_id": "string",
  "total_generated": "integer",
  "taxonomy_images": [
    {
      "domain_id": "string",
      "domain_name": "string",
      "image_b64": "string (base64-encoded SVG)"
    }
  ]
}


Frontend Notes:
Render using data:image/svg+xml;base64,${image_b64}
GET /taxonomy-image/{domain_id}
Purpose: Retrieve taxonomy SVG directly.
Response: SVG image
Frontend Notes:
Use directly in <img src="/taxonomy-image/{domain_id}">

POST /paraphrase/
Port 9090
Description
Paraphrases the given text using the T5 model and stores the input & output in MongoDB.
Request
Method: POST
 Endpoint: /paraphrase/
Body (JSON)
{
  "text": "string"
}

Response (200 OK)
{
  "input": "string",
  "paraphrase": "string"
}


Error Handling
Response:
{
  "error": "string (error message)"
}


Status Codes:
200: Success
404: Not Found
500: Server Error
